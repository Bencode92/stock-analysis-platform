Méthodologie & Transparence

Architecture du moteur de sélection et d'optimisation

Scoring fondamental

  • Buffett Score v3.1 — 6 critères pass/fail (ROE consistant, ROIC moat, levier, FCF yield, PE, expansion du moat)
  • Quality Score v2 — 5 dimensions peer-relative (Quality, Safety, Value, Growth, Momentum)
  • 4 profils sectoriels : FIN, YIELD, TECH, DEFAULT
  • Global-Adjusted Peer Scoring (PGR)
  • Payment networks reclassifiés (V, MA → DEFAULT)

Optimisation

  • Markowitz mean-variance via SLSQP
  • Fallback heuristique si contraintes incompatibles
  • 7 régimes de marché (RADAR déterministe)
  • Contraintes réglementaires (max crypto, min diversification)
  • Backtest 90 jours pour validation

Risques principaux

  • Perte en capital possible
  • Risque de marché (volatilité, corrélations)
  • Risque de change (actifs non-EUR)
  • Risque de liquidité (small caps, crypto)
  • Biais de survivorship (univers actuel)

Sources & limites

  • Prix : Twelve Data API (J-1)
  • Fondamentaux : CSV enrichis (ROE, ROIC, D/E, FCF, marges)
  • Macro : FRED API + scraping
  • Commentaires : GPT-4o-mini (sanitisé)
  • Scoring non backtesté sur forward returns
